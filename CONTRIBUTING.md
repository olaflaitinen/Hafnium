# Contributing to Hafnium

We welcome contributions to the Hafnium project. Please read this document before making any changes.

## Code of Conduct

All contributors must adhere to the [Code of Conduct](CODE_OF_CONDUCT.md).

## Development Process

1.  Fork the repository.
2.  Create a branch for your feature or fix.
3.  Write code and tests adhering to the project style.
4.  Run local tests and linting (./gradlew test spotlessCheck).
5.  Submit a Pull Request.

## Style Guide

-   **Java**: Google Java Style.
-   **Python**: Black and Ruff.
-   **Documentation**: Markdown, formal tone, no emojis.
-   **Commits**: Conventional Commits format.

## Pull Request Process

1.  Ensure all CI checks pass.
2.  Update documentation for any API changes.
3.  Add unit and integration tests.
4.  Request review from a maintainer.
