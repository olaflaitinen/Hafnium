## Getting Help

### GitHub Discussions

For general questions, ideas, and community discussions, use GitHub Discussions:

- **Q&A**: Ask questions about using Hafnium
- **Ideas**: Propose new features or improvements
- **Show and Tell**: Share how you are using Hafnium

### GitHub Issues

For bug reports and feature requests, use GitHub Issues:

- Use the bug report template for bugs
- Use the feature request template for new features
- Search existing issues before creating new ones

### Documentation Issues

For documentation improvements:

- Create an issue with the `documentation` label
- Or submit a pull request with the improvement

## Issue Guidelines

### Bug Reports

When reporting bugs, include:

1. **Version**: Hafnium version and environment details
2. **Steps to Reproduce**: Clear steps to reproduce the issue
3. **Expected Behavior**: What you expected to happen
4. **Actual Behavior**: What actually happened
5. **Logs**: Relevant log output (sanitized of sensitive data)
6. **Environment**: Operating system, Docker version, etc.

### Feature Requests

When requesting features, include:

1. **Use Case**: The problem you are trying to solve
2. **Proposed Solution**: Your suggested approach
3. **Alternatives**: Other approaches you considered
4. **Additional Context**: Any relevant background

## Response Times

We aim to respond to issues within the following timeframes:

| Issue Type | Initial Response |
|------------|------------------|
| Security vulnerabilities | 48 hours |
| Critical bugs | 3 business days |
| Other bugs | 7 business days |
| Feature requests | 14 business days |

Note: These are targets, not guarantees. Response times may vary based on maintainer availability.

## Commercial Support

For organizations requiring commercial support, service level agreements, or professional services, please contact the maintainers directly.

## Community Guidelines

When seeking support:

1. Be respectful and professional
2. Provide complete information
3. Search existing resources first
4. Follow up on your issues
5. Share solutions that work for you
